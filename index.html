☁️ Backup Otomatis 3-Harian ke Google Drive — Cara Setup (sekali saja)
Sistem ini sudah otomatis membuat file backup dan mengunduhnya tiap 3 hari sekali begitu ada yang login (tidak perlu klik apapun). Supaya file itu otomatis masuk ke Google Drive akun acikari74@gmail.com, arahkan folder unduhan browser ke folder Google Drive Desktop, satu kali saja per komputer:
  1. Install Google Drive untuk Desktop di komputer ini, login pakai acikari74@gmail.com (kalau belum ada).
  2. Biarkan Drive membuat folder sync, misalnya G:\My Drive atau ~/Google Drive. Buat subfolder baru di dalamnya, misal "Backup BRILink".
  3. Di browser (Chrome/Edge): buka Settings → Downloads → ubah lokasi unduhan ke folder "Backup BRILink" tadi, dan aktifkan "Ask where to save each file" dimatikan (off) supaya tidak perlu konfirmasi tiap kali.
  4. Selesai. Mulai sekarang, tiap file backup yang otomatis ter-download akan langsung ke-sync ke Google Drive dalam hitungan detik — bisa dicek langsung dari drive.google.com pakai akun acikari74@gmail.com.
Catatan jujur: aplikasi ini file HTML biasa, bukan aplikasi server — jadi tidak bisa "login ke Google" atau upload langsung dari dalam aplikasi (Google memang membatasi ini demi keamanan). Trik folder-sync di atas adalah cara paling praktis yang benar-benar berfungsi tanpa perlu setup server tambahan.
🌐 Sinkronisasi Multi-Device & Layar Customer (opsional)
Secara default, data (transaksi, member, kas/saldo, dll) tersimpan di masing-masing komputer sendiri-sendiri — tidak nyambung satu sama lain. Aktifkan Firebase di bawah ini untuk 2 manfaat sekaligus (gratis, butuh internet):
1️⃣ Login akun cabang yang sama di device manapun (komputer/tablet lain) otomatis melihat data yang sama, tersinkron real-time.
2️⃣ Layar Customer bisa dipakai di tablet/HP terpisah, bukan cuma jendela kedua di komputer yang sama.
📋 Cara bikin akun Firebase gratis (sekali saja, ± 5 menit, semua klik-klik saja)
  1. Buka console.firebase.google.com, login pakai akun Google (boleh pakai acikari74@gmail.com).
  2. Klik "Add project" / "Tambah project" → kasih nama bebas misal "BRILink" → lanjut terus sampai selesai (boleh matikan Google Analytics, tidak perlu).
  3. Di menu kiri, klik Build → Realtime Database → klik "Create Database" → pilih lokasi server (bebas, mis. Singapore) → pilih mode "Start in test mode" → Enable.
  4. Klik ikon ⚙️ (Project settings) di pojok kiri atas → scroll ke bawah ke bagian "Your apps" → klik ikon { } (Web) → kasih nama app bebas → Register app.
  5. Akan muncul kode berisi objek firebaseConfig = {...} — copy semua isi kurung kurawal {...} itu (termasuk kurung kurawalnya).
  6. Tempel di kotak di bawah ini, lalu klik "Simpan Konfigurasi".
⚠️ Sudah pernah setup Firebase sebelumnya? Wajib update Rules-nya dulu

Fitur sinkronisasi data ini pakai "alamat" baru di database (brilink_sync) selain yang lama (brilink_customer). Buka Firebase Console → Realtime Database → tab Rules → ganti isinya jadi:

{
  "rules": {
    "brilink_customer": { ".read": true, ".write": true },
    "brilink_sync": { ".read": true, ".write": true },
    "$other": { ".read": false, ".write": false }
  }
}

Klik "Publish" setelah diganti. Kalau baru pertama kali setup Firebase, langsung pakai rules ini dari awal (tidak perlu langkah tambahan).
